Cinematic / Virtual Production Reference

Story-to-Screen Production Pipeline

Daz Studio→Blender→Unreal Engine 5→Render & Post
The one principle

Separate “asset-library prep” (done once per asset) from “shot assembly” (done once per shot). Finish each character once as a self-contained rig — with hand sockets and a pose library — so assembling a shot is just: place → snap prop to socket → pick pose → light → shoot. A held prop bound to the hand bone inherits the wrist angle automatically, so it never has to be hand-placed.

Rules that keep it clean

◆ Held props: socket, don’t bake

Default — attach held props to a hand socket in Unreal. Keeps them swappable; the pose transfers and the prop follows the wrist automatically.

Bake-at-source (skin the prop into the mesh in Daz/Blender) only for a hero prop permanently fused to one character — e.g. a knight whose sword is skinned into his mesh, and is therefore always perfect.

◆ Standardize + retarget

One canonical skeleton per character, plus UE’s IK Rig / Retargeter, means poses and animations are shared across characters with different skeletons — Daz figures can use mocap and marketplace anims.

◆ The single change that prevents the “sword” problem

Set a hand socket on the character’s skeleton in Phase 2, give the prop a grip-pivot in Phase 3, and in the shot just attach-to-socket. Then the Blender/Daz wrist angle drives the blade automatically — no manual prop-nudging, correct from every camera angle.
